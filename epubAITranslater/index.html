面向小说、文档、长篇内容的 EPUB 翻译工作流：支持 OpenAI / DeepSeek、术语表约束、断点续跑、QA 报告与 Windows 一键安装。 默认目标语言为简体中文，强调忠实直译与结构完整性。

核心功能

双阶段翻译

支持 draft + revise 流程，可分别指定 provider/model，平衡速度、质量与成本。

可视化配置

在 GUI 内直接编辑配置（style、分段、重试、QA 阈值），无需手写 YAML。

术语一致性

支持术语表命中优先与 force 约束，减少专有名词漂移。

断点续跑

缓存命中自动跳过已翻译段，适合长书翻译中断后恢复。

QA Gate

自动检查空译文、占位符一致性、链接与 TOC 完整性，输出 qa_report 与 qa_summary。

术语自动生成

可从 EPUB 抽取候选术语，并使用 AI 批量补全空 target。

常见问题

QA error 是什么？

表示翻译流程完成，但质量检查出现错误项（例如占位符不一致、译文留原文、结构问题）。可根据 artifacts 目录中的 qa_summary.md 与 qa_report.json 定位问题。

会自动保存 API Key 吗？

不会。界面中填入的 Key 仅作用于当前进程，不会自动写入持久配置文件。

是否只能翻译成中文？

当前版本目标语言固定为简体中文（zh-Hans），并围绕中文输出质量做了针对性约束。
